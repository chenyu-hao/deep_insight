## 🎤 讲稿

---

### 开场 (Slide 1-2)

大家好，今天我想分享一下 GlobalInSight 项目最近的一个重要升级——ClawdBot 集成。

在介绍新功能之前，我想先聊聊我们之前遇到的问题。GlobalInSight 是一个舆情分析系统，它能自动爬取多平台数据、进行多角度分析、生成小红书风格的文案和配图。功能很强大，但有一个问题：**用起来太麻烦了**。

用户需要打开浏览器、进入网页、输入话题、点击分析按钮，然后盯着进度条等 10 分钟。分析完了还要点击查看结果、确认发布。整个过程需要十几次点击，而且中间不能离开，否则就不知道进度到哪了。

---

### 解决方案 (Slide 3-5)

所以我们引入了 ClawdBot。ClawdBot 是一个支持 MCP 协议的 AI 助手，我们为 GlobalInSight 开发了一个 Opinion MCP 服务，让用户可以通过自然语言对话来完成所有操作。

现在用户只需要说"帮我分析张学友拒四大天王合体"，ClawdBot 就会自动启动分析任务，并且每隔一分钟汇报进度。分析完成后，它会展示完整的文案预览，用户确认后说"OK，发布"，就自动发布到小红书了。

整个过程从十几次点击变成了三句对话，而且用户不需要盯着屏幕，可以去做其他事情，ClawdBot 会主动通知进度。

---

### 技术实现 (Slide 6-8)

从技术角度来说，我们使用了 MCP 协议。MCP 是 Anthropic 推出的 Model Context Protocol，它定义了 AI 助手和外部工具之间的标准通信方式。

我们开发的 Opinion MCP Server 运行在 18061 端口，提供了 8 个工具：启动分析、查询状态、获取结果、修改文案、发布内容、获取热榜等。ClawdBot 通过 SSE 连接和 JSON-RPC 调用来使用这些工具。

选择 MCP 协议的好处是标准化和可扩展。任何支持 MCP 的客户端都可以接入我们的服务，而且可以和其他 MCP 工具组合使用。

---

### 效果展示 (Slide 9)

实际效果非常明显。完成一次完整的分析发布流程，从之前的 15 次以上点击，减少到现在的 3 句对话。学习成本从需要熟悉界面变成了零学习——用户只需要用自然语言描述需求就行。

更重要的是用户体验的提升。以前用户需要在浏览器和其他工作之间来回切换，现在可以在同一个对话窗口里完成所有操作，工作流更加连贯。

---

### 总结 (Slide 10-11)

最后总结一下。ClawdBot 集成带来的核心改变是：从"打开网页、点击按钮、等待、查看结果"这种传统的 GUI 交互，变成了"说一句话、自动完成、收到通知"的对话式交互。

我们的目标是让 AI 工具真正融入工作流，而不是成为额外的负担。用户不应该需要学习复杂的界面，也不应该被迫盯着进度条。AI 助手应该像一个真正的助手一样，理解用户的意图，自动完成任务，并在合适的时候汇报结果。

这就是 ClawdBot 集成给 GlobalInSight 带来的改变。谢谢大家！
