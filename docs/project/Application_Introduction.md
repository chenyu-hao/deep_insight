# 应用介绍（用于导出 PDF）

## 1. 灵感来源与解决的问题
随着热点传播速度加快，信息来源碎片化、平台叙事差异扩大，传统“只看单平台热搜/热榜”的方式很难：
- 快速确认一个热点是否跨平台扩散；
- 理解不同平台对同一事件的关注点差异；
- 在有限时间内生成可用于内容创作的结构化洞察与配图素材。

本项目将“多平台热榜采集 + 对齐聚类 + 多Agent分析/生成”整合为一个可交互系统，降低热点分析与内容生产成本。

## 2. 目标用户画像
- **内容创作者**：需要快速选题、产出观点与配图（小红书/短视频/公众号等）。
- **舆情分析学习者/研究者**：希望查看热点的扩散态势、分歧点、生命周期判断。
- **评委/导师/展示场景**：需要可解释、可演示的“Agent 系统工作过程”。

## 3. 核心功能列表
- **热榜聚合与平台筛选**：全榜刷新后本地按平台过滤（含 HN/贴吧等）。
- **跨平台对齐聚类（Aligned Clusters）**：将不同平台同一热点聚为一条，提供平台证据与热度对比。
- **热点演化解读卡（Agent 生成）**：对单条热点按需调用 LLM，生成生命周期/扩散摘要/分歧点/观察点，并展示可追溯信息。
- **多Agent工作流（SSE 实时展示）**：Crawler → Reporter → Analyst → Debater → Writer → Image Generator，前端实时显示过程。
- **生成式数据卡（DataView）**：对比/情绪/关键词/平台热度等图表生成与展示，并展示 Agent 生成信息与片段。
- **AI 配图生成与预览**：生成组图，支持预加载、切换与导出。

## 4. 技术架构简介
### 前端
- Vue 3 + Vite + TailwindCSS
- Pinia 状态管理
- 图表/可视化：Chart.js（用于数据卡）

### 后端
- FastAPI（REST + SSE）
- 多Agent工作流（LangGraph / LLM 调用封装）
- 热榜采集与对齐聚类：
  - TopHub 聚合热榜
  - Hacker News 热榜（Firebase API）
  - 对齐聚类 + 历史信号（增速/新增/热度变化）
- 缓存与性能：
  - 多键缓存（内存+文件）
  - SWR（stale-while-revalidate）：先返回缓存，后台刷新重建
