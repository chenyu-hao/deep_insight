# GlobalInSight 项目优化 - 分阶段产品需求文档

本文档概述了针对小红书创作者优化 GlobalInSight 的分阶段方案，旨在实现 5 分钟内产出可发布内容的流程。

## 项目目标
重新聚焦应用，服务于需要快速判断热点价值并产出内容的小红书创作者/运营人员。
**价值主张**：“帮助用户快速理解时事热点的核心结论与证据，创作者可省时省力完成选题与发布。”

## 第一阶段：核心逻辑与内容结构（后端重点）
**目标**：确保 AI 输出符合新的“结论 + 证据 + 解读”结构以及“叙述性”风格。

### 1.1 提示词工程（Prompt Engineering）优化
- **目标文件**: `app/services/workflow.py`
- **需求**: 更新系统提示词，使其扮演“小红书内容策划专家”。
- **结构**:
  - **标题**: 12-18 字，含 emoji，吸引眼球但客观。
  - **正文**:
    - **结论**: 1-2 句，结果前置。
    - **主体**: 叙述性段落，融合事件 + 证据 + 数据（拒绝列表罗列！）。
    - **深度解析**: 1-2 句犀利点评/底层逻辑分析。
    - **互动**: 1 个提问。
    - **标签**: 3-6 个高频标签。
- **约束**: 禁止使用“首先、其次、最后”等连接词。保持高信息密度。

### 1.2 数据模型对齐
- **目标**: 后端响应结构
- 确保 API 返回结构化数据片段以支持卡片生成，或者确保“主文案”与元数据清晰分离。
- **关键字段**: `topic`, `evidence`, `platform_stats`, `time_window` (输入); `title`, `main_copy`, `tags` (输出)。

---

## 第二阶段：视觉基础与标题卡（前端重点）
**目标**：启用“标题卡”输出模式及基础 UI 切换。

### 2.1 UI 布局更新
- **目标文件**: `src/views/HomeView.vue`
- **需求**: 在预览区域增加切换/Tab 机制。
  - **选项**: “AI 配图”（默认） vs “标题卡”。

### 2.2 标题卡实现
- **目标**: 新组件 `src/components/TitleCard.vue` (或在 HomeView 内联)
- **设计**:
  - **大标题**: 来自 `analysisStore.insightTitle` 或 `xhsPreview.title`。
  - **副标题**: 来自 `analysisStore.insightSubtitle` 或衍生。
  - **样式**: Tailwind CSS，圆角卡片，半透明背景，高级感视觉。
  - **元素**: 大号文字 + Emoji + 一句数据亮点。

### 2.3 视觉切换逻辑
- **需求**: 当选择“标题卡”时，隐藏特定的 AI 生图提示/加载器，显示卡片渲染区域。
- **状态**: `imageMode` ('ai' | 'card')。

---

## 第三阶段：高级视觉与导出
**目标**：完善卡片套件（解读卡/数据卡）并确保导出流程通畅。

### 3.1 解读卡
- **需求**: 视觉卡片，将主文案拆解为：
  - 1 句结论
  - 3 条证据点
  - 1 句解释

### 3.2 数据卡
- **需求**: 数据的极简视觉表达。
  - **类型**: 饼图或条形图 (Canvas/SVG)。
  - **内容**: 平台热度对比 或 关键词 Top 5。
  - **兜底**: 若数据不足，使用文本排名的形式。

### 3.3 导出优化
- **目标**: `HomeView.vue` 中的导出功能
- **需求**: `exportAllImages` 必须支持使用 `html2canvas` 捕获当前可见的卡片（标题/解读/数据），同时也支持导出 AI 图片。

---

## 第四阶段：打磨与性能
- **目标**: 总生成时间 < 5 分钟。
- **任务**:
  - 验证端到端延迟。
  - UX 微交互（卡片的加载状态）。
  - 移动端响应式验证（预览区域）。
